# LifeDash 登录注册模块需求文档

## 一、模块概述

| 项目 | 说明 |
|------|------|
| 模块名称 | 用户认证模块（Auth Module） |
| 功能范围 | 用户注册、用户登录、密码安全、Token 认证 |
| 技术栈 | FastAPI + SQLAlchemy + MySQL + JWT + bcrypt |
| 目标 | 前端后端联调通过，账号数据持久化到数据库 |

---

## 二、数据库模型（基于现有模型扩展）

### 2.1 用户表（users）

在现有 `User` 模型基础上确认/补充字段：

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | Integer | PK, Auto Increment | 用户唯一标识 |
| `username` | String(50) | Unique, Not Null | 登录账号，支持字母数字下划线 |
| `email` | String(100) | Unique, Not Null | 邮箱，可用于找回密码 |
| `password_hash` | String(255) | Not Null | bcrypt 哈希后的密码 |
| `created_at` | DateTime | Server Default = NOW() | 注册时间 |
| `updated_at` | DateTime | Server Default = NOW(), On Update | 最后更新时间（新增） |

### 2.2 新增 updated_at 字段（模型更新）

```python
# models.py 更新
from sqlalchemy import Column, Integer, String, DateTime
from sqlalchemy.sql import func
from database import Base

class User(Base):
    __tablename__ = "users"

    id = Column(Integer, primary_key=True, index=True)
    username = Column(String(50), unique=True, nullable=False)
    email = Column(String(100), unique=True, nullable=False)
    password_hash = Column(String(255), nullable=False)
    created_at = Column(DateTime(timezone=True), server_default=func.now())
    updated_at = Column(
        DateTime(timezone=True), 
        server_default=func.now(), 
        onupdate=func.now()  # 更新时自动刷新
    )
```

> ⚠️ 修改模型后需重新执行 `Base.metadata.create_all()` 或重建表。

## 三、API 接口设计

### 3.1 接口清单

| 方法 | 路径 | 功能 | 认证要求 |
|------|------|------|----------|
| `POST` | `/api/auth/register` | 用户注册 | 无需认证 |
| `POST` | `/api/auth/login` | 用户登录 | 无需认证 |
| `GET` | `/api/auth/me` | 获取当前用户信息 | 需要 JWT Token |
| `POST` | `/api/auth/logout` | 用户登出（可选） | 需要 JWT Token |

### 3.2 请求/响应规范

#### 【注册】POST /api/auth/register

**请求体：**
```json
{
  "username": "alice2024",
  "email": "alice@example.com",
  "password": "MyP@ssw0rd123"
}
```

**字段校验规则：**

| 字段 | 规则 |
|------|------|
| `username` | 3-20 位，字母开头，允许字母/数字/下划线 |
| `email` | 标准邮箱格式 |
| `password` | 8-32 位，至少包含字母和数字 |

**成功响应（201）：**
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "id": 1,
    "username": "alice2024",
    "email": "alice@example.com",
    "created_at": "2024-01-15T08:30:00+08:00"
  }
}
```

**错误响应示例：**
```json
{
  "code": 400,
  "message": "用户名已存在"
}
```

---

#### 【登录】POST /api/auth/login

**请求体：**
```json
{
  "username": "alice2024",
  "password": "MyP@ssw0rd123"
}
```

**成功响应（200）：**
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "token_type": "bearer",
    "expires_in": 604800,
    "user": {
      "id": 1,
      "username": "alice2024",
      "email": "alice@example.com"
    }
  }
}
```

> Token 有效期：7 天（604800 秒）

**错误响应示例：**
```json
{
  "code": 401,
  "message": "用户名或密码错误"
}
```

---

#### 【获取当前用户】GET /api/auth/me

**请求头：**
```
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
```

**成功响应（200）：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "username": "alice2024",
    "email": "alice@example.com",
    "created_at": "2024-01-15T08:30:00+08:00"
  }
}
```

## 四、技术实现要点

### 4.1 密码安全

| 要求 | 实现 |
|------|------|
| 禁止明文存储 | 使用 `bcrypt` 哈希密码 |
| 哈希强度 | bcrypt rounds = 12 |
| 校验方式 | 使用 `bcrypt.checkpw()` 比对 |

```bash
# 依赖
pip install passlib[bcrypt]
```

### 4.2 JWT Token 设计

| 配置项 | 值 |
|--------|-----|
| 算法 | HS256 |
| 密钥 | 环境变量 `JWT_SECRET_KEY`（生产环境 32 位以上随机字符串） |
| 有效期 | 7 天 |
| Payload 包含 | `user_id`, `username`, `exp` |

### 4.3 统一响应格式

所有接口返回统一结构：
```python
{
    "code": 200,      # 业务状态码，200 成功
    "message": "xxx", # 提示信息
    "data": {}        # 业务数据，可为 null
}
```

### 4.4 异常处理

| 场景 | HTTP 状态码 | 业务码 | 提示信息 |
|------|-------------|--------|----------|
| 用户名已存在 | 400 | 400 | 用户名已被注册 |
| 邮箱已存在 | 400 | 400 | 邮箱已被注册 |
| 参数校验失败 | 422 | 422 | 字段格式错误提示 |
| 用户名/密码错误 | 401 | 401 | 用户名或密码错误 |
| Token 无效/过期 | 401 | 401 | 登录已过期，请重新登录 |
| 服务端错误 | 500 | 500 | 服务器内部错误 |

## 六、验收标准

| 检查项 | 通过标准 |
|--------|----------|
| 注册功能 | 输入合法参数 → 数据库新增用户记录 → 密码为哈希值 |
| 重复注册 | 同用户名/邮箱 → 返回明确错误提示，不重复入库 |
| 登录功能 | 正确凭证 → 返回有效 JWT Token |
| 错误登录 | 错误凭证 → 返回 401，不暴露具体是哪个字段错误 |
| Token 认证 | 携带有效 Token → 成功访问 `/api/auth/me` |
| Token 失效 | 无效/过期 Token → 返回 401 |
| 数据库验证 | 直接查询 MySQL，确认数据正确存储 |

## 七、安全建议

1. **JWT 密钥**：使用环境变量，生产环境至少 32 位随机字符串
2. **HTTPS**：生产环境必须启用，防止 Token 被截获
3. **密码策略**：前端后端都做校验，后端校验为最终防线
4. **频率限制**：登录接口建议加防暴力破解（如 5 次错误锁定 15 分钟）
5. **日志记录**：记录登录失败行为，便于安全审计
